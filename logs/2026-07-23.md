# Dev Log — 2026-07-23

## Private usage dashboard (`stats.html`)

Day-after-launch: the `events` collection is populated (real gameplay logging), but the Firestore console is a poor way to read it — no aggregate view, painful on mobile. Built a self-contained dashboard so launch metrics are glanceable.

### New file: `stats.html`
- Single self-contained page (no build), matches the app's dark/fire theme + Barlow fonts. Mobile-first (owner views on phone). `noindex,nofollow`.
- **Auth-gated:** Firebase Auth Google sign-in; shows a sign-in gate until authenticated. Real protection is the Firestore rule (below) — the page just reflects it.
- **Reads (no composite indexes, respects rules):**
  - Headline counts via `getCountFromServer` aggregation (single-field `type ==` filters): role_selected, scenario_started, question_answered, scenario_completed, feedback_submitted.
  - Breakdown by reading `scenario_completed` docs (single filter) → completions by scenario, by role, avg score.
  - `scenario_started` docs → per-scenario completion rate.
  - `orderBy createdAt desc limit 20` → recent activity feed.
- **Renders:** hero completion-rate card, stat grid, by-scenario table (started/completed/rate), completions-by-role bars, recent-activity list. Refresh + sign-out.
- **Graceful denial:** on `permission-denied` it surfaces the exact signed-in email so the owner knows which address to whitelist.
- Syntax-checked (`node --check`) — OK.

### `firestore.rules`
- `events` was read-denied (`allow read...: if false`). Split off `update, delete: if false` and added `allow read` limited to an authenticated, email-verified owner in an allowlist (currently `bennyphipps@gmail.com`). Writes unchanged (still public create-only, same validation).
- Not auto-deployed (no Firebase CLI) — must be pasted into Console → Firestore → Rules → Publish.

### One-time console setup required for it to work
1. Firebase → **Authentication → Sign-in method → enable Google** (set support email).
2. Firebase → **Authentication → Settings → Authorized domains → add `whatadisaster.uk`**.
3. Firebase → **Firestore → Rules → paste updated `firestore.rules` → Publish**.
4. If signing in with a different Google account than `bennyphipps@gmail.com`, add that email to the rule's allowlist (the dashboard shows the exact email if denied).

Then: `https://whatadisaster.uk/stats.html` → Sign in with Google → dashboard loads. Uses existing historical events (not just future ones).

**Files changed:** new `stats.html`; `firestore.rules`. On branch `claude/site-usage-monitoring-sasyjf`.
