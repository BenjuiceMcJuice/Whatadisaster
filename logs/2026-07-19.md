# 2026-07-19

## UI/UX mobile overhaul — Phase 2 (P1) game-screen density shipped

Built Phase 2 of `info/ui-ux-assessment-2026-07-12.md` — the "too busy" game screen. Branch `claude/next-tasks-6uydmp` (base `a1427c1`). Pre-edit snapshot kept in scratchpad (`index.before-p1.html`).

**B1 — visual hierarchy, decision first**
- Replaced the header + always-on 4-meter Consequence Tracker with a **slim status bar**: role · scenario · Score · Time on the left, a single compact **consequence pill** (colour dot + level word, e.g. 🟢 CONTROLLED) on the right, then the exit ✕. Pill is a real `<button>` → opens the status sheet.
- Moved **Consequence Tracker + Activated Agencies + Command Level** off the always-on flow into a new bottom-sheet **`#statusSheet`** ("Situation Status"), reusing the existing `.mo/.md` modal idiom. Two entry points, same sheet: the pill, and a quiet **"🏛 Agencies & Command Level ▸"** context bar placed *below* the decision.

**B3 — situation persists without dominating**
- Situation panel is now **collapsible** (`toggleSituation()`): full text on load, tap the header to collapse to a one-line recap. When collapsed it becomes **sticky** just under the status bar (`--ght` set from the header's measured height via `syncGht()` on load/toggle/resize), so the scenario title stays glanceable while scrolling the options. Resets to expanded on each `loadQ`.

**B2 — calm colour & motion**
- Removed the three simultaneous pulses/glow: `animation:up` off `.prl` ("DECIDE NOW" label) and `.ql.urg`, `animation:ep` off the escalation banner `.eb`. During a timed question only the pressure bar animates; untimed questions have no continuous motion.
- Recoloured the "danger" situation chips (`.chip.r`) from pure red to the warmer fire-orange accent, so red is reserved for wrong-answer reveal + the timer.
- The 4 consequence meters, previously 5px tall with 0.58rem labels, are enlarged (9px / 0.68rem) inside the sheet where there's room.

**B4 — stat-chip cap**
- `renderStatChips()` caps the situation to **3 chips + a "+N more"** button (`expandStatChips()` reveals the rest). Cuts the top-of-screen noise before the decision.

**B5 — scroll target met**
- Measured game-screen height at 390×844: **~1.03 screenfuls (Ashdown) / ~1.30 (Solstice)**, down from the assessment's ~2.6 and under the ≤1.6 target.

**E5 (opportunistic)** — added `@media (prefers-reduced-motion:reduce)` disabling embers/pulses/transitions.

**Verification** (headless Chromium, `/opt/pw-browsers`):
- Both scenarios driven start → results with **0 real JS errors** (only sandbox-blocked external fonts/Firebase/widget).
- Screenshots at 360 & 390 px: slim bar fits without overflow (role/scenario truncate with ellipsis), pill legible, chip cap + "+N more" render, collapsed situation sticks as a recap, status sheet opens with meters/agencies/command intact.
- ID uniqueness confirmed (cLife, clevel, agFire, clGold, statRow, etc. → 1 each after the move).

**Preserved:** single-file/no-build, fire-orange identity, all question content/scoring untouched.

**Still open in the overhaul:** P2 (splash CTAs above the fold, de-jargon lead, "8 roles"→9 — overlaps the shipped LinkedIn OG/title work) and P3 polish. The Solstice fire-map blob-id bug remains separately tracked.
