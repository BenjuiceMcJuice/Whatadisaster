# Dev Log — 2026-07-11

## Feedback moved onto the shared Benjuicey Apps backend

Trigger: Ben confirmed the requirement flagged 2026-07-10 (feedback must be one system across all apps, not siloed per-app) and gave access to the `benjuicey-apps` repo to resolve it. Full decision written up in `info/linkedin-launch-readiness-spec.md` under "Cross-app feedback integration."

**Decision:** apps POST to a shared Cloudflare Worker (`benjuicey-apps` repo, `benjuicey-feedback.benjuicemcjuice.workers.dev`) which writes server-side to a shared Firestore project via a service account — not a shared Firestore project apps write to directly. This app's trigram is `WDA` (already reserved in the Worker's `trigrams.ts`).

**Built on the `benjuicey-apps` side** (see that repo's own log):
- New `GET /widget.js` route on the Worker serving a reusable, themeable embeddable feedback widget (name/email/type/message form, matches the field contract `ai-literate` was already using by hand).
- Widget dispatches a `benjuiceyfeedback:submitted` window event on success so host apps can hook their own analytics.
- Added `whatadisaster.uk` / `.pages.dev` to the Worker's `ALLOWED_ORIGINS`.

**Changed in this repo (`index.html`):**
- Removed the 2026-07-10 bespoke feedback implementation: `.fb-toggle` button, `#feedbackModal`, `window.submitFeedback`, `openFeedback()`/`closeFeedback()`/`sendFeedback()`.
- Added `<script data-app-id="whatadisaster" data-accent="#FF4500" src=".../widget.js">` — same widget every other app uses.
- Kept `window.logEvent` / the `events` Firestore write path as-is (usage analytics is a separate, per-app concern from feedback) — now listens for `benjuiceyfeedback:submitted` to keep firing the `feedback_submitted` usage event.

**`firestore.rules`:** removed the `/feedback/{id}` match block — this project's rules now cover `events` only. Still not published in the Firebase console (carried over from 2026-07-10, still Ben to do).

**Docs updated:** `DEVLOG.md` (blocked section resolved, new milestone, feedback section rewritten), `CLAUDE.md` (session-start feedback review no longer queries a local Firestore collection), `info/linkedin-launch-readiness-spec.md` (status + cross-app section + stale Stage 1/2 references corrected).
