# 2026-07-15

## Timed decisions: reading grace + longer countdowns

**Trigger:** more feedback that timed ("⚡ URGENT DECISION") questions are too short — *even though* click-to-reveal shipped 2026-07-12. Root cause: the reveal gate only moved the *situation* text out from under the clock. `revealTimedQ()` renders the four option paragraphs **and** starts the countdown in the same instant, so reading the options is still fully timed. Durations were also low (6×10s, 8×12s, 1×15s, 1×25s).

**Change (`index.html`):**

1. **Reading grace in `startPressTimer`** — after reveal the pressure bar now holds full (clock frozen, no penalty) for a fixed **2.5s reading window**, with the label reading "👁 READ THE OPTIONS…", before it begins draining. Once grace elapses the existing "⚡ DECIDE NOW — Ns" countdown behaves exactly as before, including the timeout → worst-option auto-pick with doubled consequences (unchanged). One added guard line at the top of the interval body; no change to `revealTimedQ()`.

2. **All 16 `timerSecs` extended ~+50%** (per-question values kept authoritative/tunable): 10→15 (×6), 12→18 (×8), 15→22 (×1), 25→37 (×1). Net headroom per question ≈ 2.5s untimed read + a 50%-longer drain (e.g. a 12s question → 2.5s read + 18s decide).

Out of scope (deliberately unchanged): timeout severity (worst-option auto-pick + doubled `cons`). Feedback asked for more time, not softer penalties. No copy elsewhere hardcodes durations (`how-it-works.html` doesn't describe the timer).

**Verified:**
- Browser (headless Chromium): timed question gates options behind the reveal button (0 options shown pre-reveal); clicking reveal renders the options with `#pressArea.on` and the "👁 READ THE OPTIONS…" label + full bar; a timed-out question still auto-selects the worst option (`.tout`) and shows the modal.
- Deterministic tick test running the real `startPressTimer` source against a manual 100ms clock: grace holds the full bar until ~2.5s, then "⚡ DECIDE NOW — Ns" drains 95%→0%, expiry at ~2.5s+drain, worst option (pts 0) auto-picked with `timedOut:true`. PASS.
