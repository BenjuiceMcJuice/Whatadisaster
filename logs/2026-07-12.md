# Dev Log — 2026-07-12

## Click-to-reveal for timed questions

Trigger: stakeholder feedback (Clare + Jon, `info/stakeholder-feedback-june-2026.md`) — timed-question countdowns start too fast, no time to read the options first.

**Change (`index.html`):** for `timed:true` questions, `loadQ()` now renders the situation/question text with a single "👁 Reveal Options — starts the clock" button in place of the answer options. The countdown (`startPressTimer`) no longer starts on question load. A new `revealTimedQ()` function swaps in the real options and starts the timer only once the player clicks reveal. Untimed questions are unaffected — options render immediately as before.

Factored the option-button HTML building into a shared `buildOptsHtml()` helper (was inlined in `loadQ()`, now reused by both `loadQ()` and `revealTimedQ()`).

Added `.ob.reveal` CSS for the new button (centered, red-tinted to match the urgent styling).

**Verified** via local static server + browser automation: reveal button shows with options/timer hidden, clicking it renders options and starts the countdown (`pressFill` animates from 100%), and natural timeout still correctly triggers the worst-scoring option with the "TIME EXPIRED" modal.

**Not done:** same content-accuracy fixes from the stakeholder doc (TCG/hospital wording, NWFC→NRFC, crew safety conditional, ambulance surge framing, telegraphing, utilities split) — separate backlog items, working through them next.

## UI/UX assessment + build spec (mobile-focused)

Trigger: Ben asked for a full UI/UX review — worry that the app is "a bit busy" — with mobile (iOS/Android) as the primary target, and a fully-specced md to build later.

**Method:** static read of `index.html` + live render at iPhone (390×844) and reference to Android (360×800) via the pre-installed headless Chromium. Screenshotted splash / role-select / game screens to ground the density findings.

**Deliverable:** `info/ui-ux-assessment-2026-07-12.md` — severity-ranked (P0–P3), each finding with `file:line` evidence, fix, effort, and acceptance criteria; plus a phased build order and a whole-effort acceptance list.

**Key findings:**
- **P0 correctness/a11y:** pinch-zoom disabled (`user-scalable=no`, `index.html:5` + `how-it-works.html:5`, WCAG 1.4.4 fail); **duplicated DOM** — two copies each of `#game`/`#results`/`#modal`/`#fireMap` (live = first, dead = `:488/:535/:561/:435`) with clashing IDs; modal **backdrop tap advances the question** (`closeModal()` does `qIdx++`); no in-game exit/quit; role cards are `<div onclick>` not buttons; safe-area insets ignored.
- **P1 "too busy" (game screen):** ~2,160px tall per question at 390×844 (~2.6 screenfuls); six persistent panels around the decision; Agencies + Command Level sit *below* the options (passive noise); 3 things pulse at once; red used for everything. Spec proposes: slim sticky status bar + single consequence pill, demote Agencies/Command to a drawer, collapsible/persistent situation, calm colour/motion, cap stat chips → target ≤1.6 screenfuls.
- **P2 splash:** ~2,900px tall, acronym-heavy framing (WCS/RWCS/SCG/JESIP/BC) *above* the play buttons → move CTAs above the fold, framing into an accordion/briefing, de-jargon the lead. Also stale copy "8 roles" → 9 (`index.html:222`). Plus `<title>`/OG/Twitter tags (overlaps the launch spec).
- **P3 polish:** timer durations still 10–12s, `prefers-reduced-motion`, `:focus-visible`, orphaned last role card.

**Non-goals held:** single-file/no-build architecture, dark fire-orange identity, and question content/scoring all preserved. Nothing implemented yet — this is a spec.

**Tooling note:** installed `playwright` locally only to screenshot; removed `node_modules`/`package*.json` before commit so they aren't tracked.
