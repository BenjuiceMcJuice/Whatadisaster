# Dev Log — 2026-07-12

## Click-to-reveal for timed questions

Trigger: stakeholder feedback (Clare + Jon, `info/stakeholder-feedback-june-2026.md`) — timed-question countdowns start too fast, no time to read the options first.

**Change (`index.html`):** for `timed:true` questions, `loadQ()` now renders the situation/question text with a single "👁 Reveal Options — starts the clock" button in place of the answer options. The countdown (`startPressTimer`) no longer starts on question load. A new `revealTimedQ()` function swaps in the real options and starts the timer only once the player clicks reveal. Untimed questions are unaffected — options render immediately as before.

Factored the option-button HTML building into a shared `buildOptsHtml()` helper (was inlined in `loadQ()`, now reused by both `loadQ()` and `revealTimedQ()`).

Added `.ob.reveal` CSS for the new button (centered, red-tinted to match the urgent styling).

**Verified** via local static server + browser automation: reveal button shows with options/timer hidden, clicking it renders options and starts the countdown (`pressFill` animates from 100%), and natural timeout still correctly triggers the worst-scoring option with the "TIME EXPIRED" modal.

**Not done:** answer telegraphing, utilities split — separate backlog items, working through next.
